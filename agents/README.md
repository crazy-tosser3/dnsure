# Документация агента `dnsure-agent`

Этот документ описывает функциональность агента `dnsure-agent`, который предназначен для выполнения различных сетевых проверок по запросу от центрального сервера.

## Функциональность

Агент предоставляет один основной эндпоинт для запуска проверок. При старте агент генерирует или загружает свой уникальный UUID, который используется для аутентификации запросов от центрального сервера.

## Эндпоинты

### 1. Запуск проверки

*   **URL:** `/api/start_check`
*   **Метод:** `POST`
*   **Описание:** Эндпоинт для запуска одной или нескольких сетевых проверок на указанном хосте.

#### Ожидаемый запрос (Request Body - JSON)

Агент ожидает JSON-объект, соответствующий следующей структуре:

```json
{
    "host_to_check": "example.com",
    "check_type": {
        "ping": true,
        "tcp": false,
        "http": true,
        "https": true,
        "traceroute": false
    },
    "agents_location": "russia",
    "agent_uuid": "ваш_уникальный_uuid_агента"
}
```

*   `host_to_check` (str): Целевой хост или IP-адрес для проверки.
*   `check_type` (Dict[str, bool]): Словарь, где ключи — это типы проверок (`ping`, `tcp`, `http`, `https`, `traceroute`), а значения — булевы флаги, указывающие, нужно ли выполнять данную проверку.
*   `agents_location` (str): Локация агента (используется центральным сервером для выбора агентов).
*   `agent_uuid` (str): Уникальный идентификатор агента, который должен совпадать с UUID, сгенерированным агентом при запуске.

#### Пример ответа (Response Body - JSON)

В случае успешного выполнения проверок:

```json
{
    "status": "completed",
    "host_checked": "example.com",
    "results": {
        "http": 120,
        "https": 150,
        "ping": 35,
        "tcp": 80,
        "traceroute": ["192.168.1.1", "10.0.0.1", "example.com"]
    }
}
```

В случае ошибки (например, неверный UUID):

```json
{
    "detail": "Неверный UUID агента."
}
```

## Запуск агента

1.  **Установка зависимостей:**
    Убедитесь, что у вас установлен `uv`. Затем установите зависимости:
    ```bash
    uv pip install -r requirements.txt
    ```
    *(Примечание: `requirements.txt` должен содержать `fastapi`, `uvicorn`, `pythonping`, `scapy`, `requests`, `pydantic` и `dnspython`)*

2.  **Запуск:**
    Запустите агент с помощью `uvicorn`. При первом запуске будет сгенерирован файл `agent.uuid` с уникальным идентификатором.
    ```bash
    fastapi run main:app --host 0.0.0.0 --port 8001 --reload
    ```
    *   `--host 0.0.0.0`: Агент будет доступен по всем сетевым интерфейсам.
    *   `--port 8001`: Агент будет слушать на порту 8001 (может быть изменен).
    *   `--reload`: Автоматическая перезагрузка при изменении кода (удобно для разработки).

    **Важно:** Для корректной работы `pythonping` и `scapy` могут потребоваться права суперпользователя (root) для использования raw sockets. Если вы запускаете агент в Docker, убедитесь, что контейнер имеет необходимые привилегии.

3.  **UUID агента:**
    При первом запуске агент создаст файл `agent.uuid` в той же директории, где находится `main.py`. Этот UUID необходимо будет использовать при регистрации агента на центральном сервере.

    Пример содержимого `agent.uuid`:
    ```
    a1b2c3d4-e5f6-7890-1234-567890abcdef
    ```

*   **Swagger UI:** `http://<agent_host>:8001/docs`
*   **ReDoc:** `http://<agent_host>:8001/redoc`
