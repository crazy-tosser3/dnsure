# Документация API проекта `dnsure`

Этот документ содержит описание REST API эндпоинтов, реализованных в приложении.

## Модели данных (Pydantic Schemas)

### `ConnInfo`

Используется для передачи информации о подключении к агенту.

* `host` (str): IP-адрес или доменное имя.
* `port` (int): Порт для подключения.
* `password` (str): Пароль.

### `CheckInfo`

Используется для инициализации проверки.

* `host` (str): Проверяемый хост.
* `check_type` (list): Список типов проверок для выполнения.
* `agent_uuid` (str): UUID конкретного агента, на котором будет выполняться проверка.

## Эндпоинты

### 1. Регистрация агента

* **URL:** `/api/agent_add`
* **Метод:** `POST`
* **Тело запроса (JSON):** `ConnInfo`
* **Описание:** Поинт для регистрации нового агента в системе.

### 2. Удаление агента

* **URL:** `/api/agent_del`
* **Метод:** `DELETE`
* **Тело запроса (JSON):** `ConnInfo`
* **Описание:** Поинт для удаления существующего агента из системы.

### 3. Получение списка агентов

* **URL:** `/api/get_agents`
* **Метод:** `GET`
* **Тело запроса:** Нет
* **Описание:** Поинт для получения списка всех зарегистрированных агентов.

### 4. Запуск проверки

* **URL:** `/api/start_check`
* **Метод:** `POST`
* **Тело запроса (JSON):** `CheckInfo`
* **Описание:** Поинт для запуска проверки на конкретном агенте, указанном по его UUID.

## Запуск проекта

### 1. Запуск базы данных

Проект использует PostgreSQL. Для быстрого развертывания базы данных с помощью Docker выполните команду:

```bash
docker run -d \
  --name practicaDB \
  -e POSTGRES_PASSWORD=78942983 \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_DB=postgres \
  -p 5432:5432 \
  postgres:16
```

Команда для подключения к контейнеру с БД через консоль:

```bash
docker exec -it practicaDB psql -U postgres -d postgres
```

*(Примечание: если вам нужно использовать другие учетные данные, как в вашем изначальном примере, используйте `docker exec -it practicaDB psql -U SERVER -d serverDB`)*

### 2. Переменные окружения

Создайте файл `.env` в корне проекта (в одной папке с `main.py`) и добавьте в него следующие настройки подключения:

```env
POSTGRES_USER=postgres
POSTGRES_PASSWORD=78942983
POSTGRES_HOST=127.0.0.1
POSTGRES_PORT=5432
POSTGRES_DB=postgres
```

### 3.1.  Установка зависимостей и запуск сервера

В проекте используется высокопроизводительный пакетный менеджер `uv`. Убедитесь, что он установлен, затем выполните следующие команды:

```bash
# Установка зависимостей
uv pip install -r requirements.txt

# Запуск сервера с автоматической перезагрузкой при изменениях кода
fastapi run
```


### 3.2. Установка завичимостей и запуск сервера (без uv)

```bash
pip install -r venv .venv

fastapi run
```
